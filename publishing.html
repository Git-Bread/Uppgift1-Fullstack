Om Render:

Render är den webbtjänst jag använder för att ladda upp denna webbplats, varför Render? Jo för det är enkelt att använda, med stor kontroll över hur uppladdningen och formateringen skes. Plus möjligheten att aktivt hålla det syncat med ett github rep och inte behöva skapa ett 149:onde konto på webben.

Jag har använt Render under majoriteten av kursen, på grund av tre anledningar Enkelt, Gratis och fungerar. Försökte först med Vercel men det sket sig och jag blev inge visare av att försöka troubleshoota det, medans Render fungerade out of the box.


Dessutom så har den en väldigt minimalistsk UI som inte slösar din tid på fina grafiker eller 24 navigerings möjlighter, direkt när man loggar in så listar den alla ens webbplatser med statusn, regionen o.s.v. Har hittils aldrig haft något större problem med tjänsten, fast har inte betalat för den och den lär vara skit-dyr då det är en AWS-repacker som har modern UI.

Alright, slut på opinions delen, dags för faktiska fördelar nakckdelar och priser.

  • Pris:

    Render har en gratisversion med majoriteten utav funktionaliteten, fast oftast segare servrar, den som används till denna uppgift. Så Gratis +. För företag och liknande så är det ett statiskt pris per användare med "rörliga" kostnader kring RAM och liknande.

    För alla priser och nivåer, o.s.v. se Render

  • Documentation:

    Render erbjuder en expansiv dokumentation, med bred information om "uppladdnings metodik" och felsökning. Uppstår ett problem eller behöver något eftersökas kan det i stort sett alltid hittas via dokumentationen.

  • Stöd:

    Render kan "köra" en stor andel runtimes såsom Node, Docker, Python, Elexir, Ruby, Go, Rust, PHP och dessutom stöd för databaser såsom MySQL databaser. Databaser finns inte i gratis versionen.

  • Uppladdnings Arbetsättet

    För statiska webbplatser såsom dessa så loggar man in på sitt konto eller genom sitt github/google eller liknande. Varefter man väljer New -> Static Site för att skapa en statisk webbplats. Sedan så får man välja ett Git-Repo, antingen eget eller publikt, som kommer att dupliceras till Render som sedan kör det åt dig. I detta fall Så väljs det egna repot "Uppgift1-Fullstack".


    Varefter man får välja branch, root, build command och publish directory. Dessutom så kan man deklarera enviroment variables, däremot så krävs det en build directory så webbplatsen måste byggas med yarn eller något liknande, i detta fall så görs detta via parcel (npm run build -> npm run parcel).
